# Doctor's Logbook - Patient Management System

A responsive web application for doctors to manage patient records, built with Bootstrap and optimized for Mac and iPhone devices. Data is stored in Google Sheets using Google Apps Script.

## 🌐 Live Demo
Visit the live site: **https://drdr-shabeelsulaiman.github.io**

## ✨ Features

- **Responsive Design**: Optimized for Mac, iPhone, and all devices
- **Patient Management**: Add, view, search, and manage patient records
- **Google Sheets Integration**: Secure data storage using Google Apps Script
- **PDF Printing**: Generate printable patient records
- **Modern UI**: Clean Bootstrap interface with smooth animations
- **Real-time Sync**: Patient data syncs immediately to Google Sheets

## 🚀 Quick Start

1. **Visit the live site**: https://drdr-shabeelsulaiman.github.io
2. **Add your first patient**: Fill in the patient form and click "Save Patient"
3. **View records**: Browse all patient records in the table below
4. **Search patients**: Use the search box to find specific patients
5. **Print records**: Click the printer icon to generate PDF records

## 📱 Device Compatibility

- **Desktop**: Full-featured interface for Mac/Windows
- **Tablet**: Optimized layout for iPad and Android tablets
- **Mobile**: Touch-friendly interface for iPhone and Android phones
- **Print**: Optimized print layouts for medical records

## 🛠️ Technology Stack

- **Frontend**: HTML5, CSS3, JavaScript, Bootstrap 5
- **Backend**: Google Apps Script
- **Database**: Google Sheets
- **Hosting**: GitHub Pages
- **Icons**: Bootstrap Icons

## 📊 Data Management

All patient data is securely stored in Google Sheets with:
- **Automatic backups** by Google's infrastructure
- **Real-time synchronization** across devices
- **Export capabilities** to CSV/PDF formats
- **Access control** through Google account permissions

## 🔒 Security & Privacy

- **HTTPS encryption** for all data transmission
- **Google authentication** for data access
- **No local storage** of sensitive patient information
- **CORS protection** for cross-origin requests

## 📋 Patient Record Fields

- Personal Information (Name, Age, Gender, Contact)
- Visit Details (Date, Chief Complaint)
- Medical Information (Diagnosis, Treatment Plan)
- Additional Notes (Custom observations)

## 🖨️ Printing & Export

- **Patient records**: Individual patient printouts
- **Batch printing**: Multiple patient summaries
- **PDF generation**: High-quality medical documents
- **Data export**: CSV export from Google Sheets
